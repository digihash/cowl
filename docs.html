Intro

Welcome to Cowl. Cowl is an Object Oriented PHP 5 framework designed to make writing boring, generic websites less boring and generic. Created by teen super child Erik Andersson from the dark abcsesses of Sweden.

Goal

First and formost, the goal of Cowl is to stay alive as an open source project and receive a steady stream of commits improving the ecosystem that is Cowl. Secondly the goal is to make creating websites fun and challenging again. The time spent om boring things should always be at a minimun when developing with Cowl.

Cowl is not meant to be a super MVC framework, because I have never taken the time to learn what really constitutes an MVC framework. The main point is to work in real world cases. Cowl was built from needs.

Style

Sadly, Cowl can not force you to write good code (but the devs (me) will probably hate you if you're stupid). However, the damages will be kept at a minimum if you choose to follow these conventions.

Conventions

Non-believers will probably hate these conventions, and I hate non-believers.

To keep the code clean and standardized in Core Cowl developers should always follow these conventions, unless they want their hard work to get neglected, unfed, murdered and rot in code hell. Damn sadists I say.

If you find any non-conventional code in Core Cowl you are allowed to crucify the dev responsible. It is you duty as a human being to correct the atrocity.

Names

Names should follow the following conventions:

Code
Classes
PascalCased. Namespacing names ala PHP 4|5 foo_bar_MooTools is not necessary and should not be used.
Methods
camelCased verbs. Names should be descriptive and short. The less words the better.
Getters and setters should be prefixed with getFoo() respectively setFoo().
Properties
with_underscore. If a boolean value is stored is_ should be prefixed, so it can be read as "Is exists?" or "Is dirty?".
Variables
Variables follow the same conventions as class properties.
Files

Files follow special rules and therefor gets their own definition list:

Core files
Core files should be placed in the cowl/-directory with simply the name of the class contained (lowercased) as its own.
Plugins
Plugins are placed in the cowl/plugins/-directory. They can be placed in any number of sub-directories and the actual plugin filename should be prefixed with plugin.pluginname.php. Other files are ignored and can thus be used as library files.
Commands
Prefixed with command.commandname.php and commandname is of course the name of the contained command (class).

Brackets

Finally it is my turn to govern whether the newline should be placed before or after the brackets!

The newline should be placed after the closing parens and before the bracket, resulting in this spacious code:


/*
	These block-statements follow the general convention.
*/

if ( $user->is_online )
{
	// ...	
}

while ( true )
{
	// ...
}

foreach ( $users as $user )
{
	// ...
}

for ( $i = 0; $i < 100; $i++ )
{
	// ...
}

/*
	These do not.
*/

// Don't we all love exceptions
try {
	throw new Exception('Error');
} catch ( Exception $e ) {
	// ...
}

do { // do looks naked without the bracket to the right of it
	
} while ( false );

Notice the exception in the try { ... } catch () { ... }-block and do { ... } while ();.

Indentation

Indentation should almost always be a tab (\t) character. The exception is in the config.ini file, where spaces should be used for unified indentation.

Spaces

Documentation

Cowl's style of documentation follows that of NaturalDocs. For the curious, information can be found at NaturalDocs website.

Be sure to always take some time off every coding session to walk through and document your code. This step is very important. Any code commited that is not thouroghly documentated will not be accepted.
